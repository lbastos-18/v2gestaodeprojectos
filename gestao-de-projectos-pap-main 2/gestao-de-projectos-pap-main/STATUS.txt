🔒 SEGURANÇA - IMPLEMENTAÇÕES
═══════════════════════════════════════════════════════════════════════════════

[✅] Autenticação Segura
    └─ Sessões PHP com cookies HttpOnly/Secure
    └─ Sem localStorage (impossível escalar privilégios)
    └─ Timeout automático de sessão
    └─ Regeneração de ID após login

[✅] Proteção CSRF
    └─ Tokens gerados por sessão (32 bytes)
    └─ Validação obrigatória em todas operações sensíveis
    └─ Endpoint dedicado para obter token
    └─ Integração no frontend automática

[✅] Configuração Segura
    └─ Variáveis de ambiente (.env)
    └─ Sem credenciais no código
    └─ Template .env.example para documentação
    └─ .gitignore para excluir ficheiros sensíveis

[✅] Rate Limiting
    └─ 5 tentativas por 15 minutos
    └─ Proteção contra brute-force
    └─ Reset após login bem-sucedido
    └─ Configurável via .env

[✅] Validação e Sanitização
    └─ Validação de tipo (email, inteiros, strings)
    └─ Limite de comprimento para campos
    └─ Prepared statements (SQL injection prevention)
    └─ Validação MIME type de ficheiros
    └─ Tamanho máximo configurável

[✅] Proteção de Uploads
    └─ Validação MIME (application/pdf)
    └─ Nomes aleatórios e seguros
    └─ .htaccess bloqueia execução PHP
    └─ Limite de tamanho: 5 MB padrão

[✅] Autorização Server-Side
    └─ Verificação obrigatória em todas rotas
    └─ Verificação de papel (admin vs user)
    └─ Verificação de propriedade (criador)
    └─ Respostas HTTP adequadas (401, 403)

[✅] Logging e Tratamento de Erros
    └─ Logs em ficheiros (não stdout)
    └─ Mensagens genéricas ao cliente
    └─ Detalhes em desenvolvimento
    └─ Flag APP_DEBUG para controlar
